# Hanzi 项目完善总计划（先不讨论收费）

> 目标：正式版上线早期，先把「查字/发音/笔顺/字典」的核心体验做稳、做快、做耐用；待留存与口碑更稳定后，再讨论广告、买断或订阅等收费策略。

## 1) 约束与原则

- 汉字展示（楷体/笔画渲染）不改动；只做 UI 结构与视觉系统优化。
- 优先做稳定性/性能/可用性；避免早期引入影响口碑的强打扰（广告/付费墙）。
- 信息架构以“顶层目的地”而非“功能数量”为准：底部 4 Tab 起步，后续扩展不推翻结构。

## 2) 顶层信息架构（已决定：4 Tab）

- `Home`：练习主入口（主 CTA）、搜索、继续练习、后续接入「每日一字」。
- `Learn`：课程 + 进度（内部二级 Tab/Segment）。
- `Dict`：字典搜索 + 收藏/历史。
- `Me`：设置、帮助/隐私、反馈、数据管理；承载音量安全/引导页开关。

## 3) 近期优先级（建议按迭代推进）

### P0（1–2 周）：核心体验“稳 + 快”

- 练习页：底部 3 动作（发音 / 主按钮 / 字典），低频入口收拢到「设置/更多」。
- 暗黑模式：统一走 `MaterialTheme.colorScheme`，画布/卡片/文字在深色下可读且克制。
- 字典：按需加载 + Loading/失败/重试状态；避免影响练习页首轮体验。
- TTS：初始化与可用性判断完善；不可用时不崩溃、不给用户“点了没反应”的体验。

### P1（2–6 周）：更“专业感”的产品体验

- 音量安全提醒：播放前检测媒体音量阈值，提供检查/降音/取消。
- 首次引导（Onboarding）：3–4 页说明核心价值与基本用法，完成后不再打扰，可在设置里重看。
- 字典好用化：多结果列表、拼音输入、模糊匹配、与练习页联动（查词→练习→返回）。
- 工程化：拆分 `CharacterViewModel` 的职责边界，补齐关键单元测试与 Compose UI 测试。

### P2（依赖每日任务后再做）：增长与便捷入口

- 每日任务/打卡：每日一字、streak、进度模型与提醒（可选）。
- 桌面组件（Widget / Glance）：显示今日汉字与快捷入口（优先做“打开 App 并自动播放/展示”MVP）。
